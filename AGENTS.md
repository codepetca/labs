# CodePet Labs Agent Notes

Before changing UI, read `docs/ui-ux-style.md`.

Keep the public site mobile-first, minimal, and low-text. Prefer concise labels,
small cards, obvious actions, and mock-data messaging. Do not add auth,
databases, production Pika access, or real student data flows to this repo.

Run `pnpm lint` and `pnpm build` before shipping UI changes.
