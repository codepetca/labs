# AI Guidelines

AI is expected in CodePet Labs work. It should raise the quality and speed of learning, not hide weak understanding.

## Use AI For

- Exploring implementation options.
- Drafting UI copy and documentation.
- Generating first-pass code.
- Debugging with clear reproduction steps.
- Reviewing code for edge cases.
- Learning unfamiliar APIs.

## Verify Everything

Students are responsible for the final work they submit.

- Read generated code before committing it.
- Run the app locally.
- Run lint and build checks.
- Test the main user flow.
- Remove unused code and fake complexity.
- Be able to explain important decisions.

## Do Not Use AI For

- Inventing fake production behavior.
- Adding auth, database, or integrations that are not needed.
- Copying private code or data into external tools.
- Submitting work you do not understand.

## Practical Standard

Good AI-assisted work is clear, tested, and explainable. If AI creates something confusing, simplify it before asking for review.

For UI work, follow [UI/UX Style Guide](ui-ux-style.md): mobile-first, minimal, low-text, and mock-data-first.
