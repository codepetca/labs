# Project Ideas

Labs projects should orbit Pika without requiring production access.

## Good Project Shapes

- Mock-first dashboards.
- Classroom workflow prototypes.
- Student progress visualizations.
- Lightweight learning companions.
- Teacher utility tools.
- UI experiments for future Pika surfaces.
- Data import or export sketches using fake files.
- Documentation and onboarding tools.

## Starter Ideas

- CodepetPal XP and streak prototype.
- Gradex dashboard with mock class data.
- TapCheck attendance check-in flow.
- Pika polling playground.
- Student portfolio generator for project summaries.
- Teacher feedback queue with mock submissions.
- Classroom activity timeline using local JSON.
- Design system playground for Pika-adjacent components.

## Avoid For Now

- Production database writes.
- Real student data.
- Real authentication.
- Payment flows.
- Admin tools that imply production authority.
- Anything that requires direct Pika core changes.

The best first projects are small enough to demo in a week and useful enough to teach real product judgment.
