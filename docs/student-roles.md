# Student Roles

Roles in Codepet Labs describe the kind of contribution a student is practicing. They are not job titles.

## Builder

Builds small features, prototypes, UI screens, and workflows. A builder should learn to keep scope small, test changes locally, and explain tradeoffs.

## Product Explorer

Clarifies the problem, writes simple specs, compares options, and helps decide what the prototype should prove.

## UI Prototyper

Works on interface layout, visual polish, responsive behavior, and demo-ready screens.

## QA Reviewer

Tests flows, catches regressions, writes reproduction notes, and checks whether a demo is honest about mocked behavior.

## Documentation Lead

Improves READMEs, setup notes, build logs, and portfolio-facing explanations.

Students may move between roles. The goal is to build range while producing clear, useful work.
