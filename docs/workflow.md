# Workflow

Most Labs work is async through GitHub and Discord.

## Weekly Sync

Weekly syncs are for:

- Demos of working slices.
- Direction-setting.
- Blockers.
- Scope decisions.
- Next steps.

They are not long status meetings. A good sync answers: what changed, what works, what is blocked, and what happens next.

## GitHub Flow

- Each project should live in its own repo when possible.
- Use one worktree per branch. See [`docs/dev-workflow.md`](./dev-workflow.md)
  for the Labs worktree convention.
- Work should happen through branches and pull requests.
- Pull requests should explain the change, the test path, and any known gaps.
- Small PRs are better than large unclear drops.
- Screenshots or short videos are useful for UI work.

## Discord Flow

- Use Discord for quick blockers, decisions, and links to work.
- Keep important decisions in GitHub issues, PRs, or docs so they are not lost.
- Ask for help early when blocked.

## Demo Standard

A demo does not need to be finished. It should be honest. Show what works, what is fake, what data is mocked, and what decision the demo is meant to answer.
